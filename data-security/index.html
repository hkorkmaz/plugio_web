Data Security Statement

Effective Date: February 6, 2026

1. Overview

At Plugio, data security is fundamental to how we build our products. Our Jira visualization plugin (gauge, counter, and progress bar gadgets) has been designed with a security-first approach: we do not collect, store, process, or transmit any data. This document outlines our security posture and architecture.

2. Architecture: No Data Collection

The Plugio plugin operates entirely within your Atlassian Jira environment. Our architecture ensures:

  • No external servers: Plugio does not operate any backend servers, databases, or cloud infrastructure that receives your data.
  • No network calls: The Plugin does not make any network requests to Plugio-owned or third-party endpoints.
  • Client-side rendering: All visualization rendering occurs in the user's browser within the Jira application context.
  • Jira-native data access: The Plugin reads data exclusively through Atlassian's standard gadget APIs, the same mechanism used by Jira's built-in gadgets.

3. Data Flow

The data flow within our Plugin is straightforward and contained:

  1. The user configures a gadget on their Jira dashboard (e.g., selecting a JQL filter or a project metric).
  2. The gadget configuration is stored by Jira's own dashboard system â€” not by Plugio.
  3. When the dashboard loads, the Plugin reads the configured data from Jira using standard Atlassian APIs.
  4. The Plugin renders the data as a visual widget (gauge, counter, or progress bar) directly in the browser.

At no point does data leave your Jira instance through the Plugin.

4. What We Do Not Have

  • No databases or data stores of any kind.
  • No server infrastructure that processes user data.
  • No analytics or telemetry collection.
  • No cookies, local storage usage, or browser fingerprinting.
  • No access tokens, API keys, or credentials stored outside of Jira.
  • No logging of user activity or Jira data.

5. Atlassian Security Compliance

As an Atlassian Marketplace app, Plugio adheres to Atlassian's security requirements for marketplace vendors. The Plugin is distributed through the Atlassian Marketplace and is subject to Atlassian's review processes.

6. Your Data Stays with Atlassian

Your Jira data remains under the control of your Atlassian instance at all times. The security of your underlying Jira data is governed by Atlassian's own security practices and your organization's Jira administration policies. Plugio does not alter, affect, or introduce any additional risk to your Jira data security posture.

7. Vulnerability Reporting

If you believe you have discovered a security vulnerability in the Plugio plugin, please report it to us promptly. We take all reports seriously and will investigate and respond as quickly as possible.

Email: security@plugio.io
